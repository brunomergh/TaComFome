Tacomfome

Exercício para implementar os padrões estruturais. Foram aplicados o padrao Flyweight, Decorator, Facade, Adapter, Bridge, Composite

Adapter - Adaptador (Classe) Converte a interface de uma classe em outra interface que clientes possam utilizar. Compatibiliza classes, permitindo que trabalhem em conjunto.

Bridge - Ponte (Objeto) Desacopla uma abstração de sua implementação, de modo que as duas possam variar independentemente. Composite - Composição (Objeto) Compor objetos em estruturas de árvores para representar hierarquias parte-todo. Permitir que clientes tratem de modo uniforme objetos individuais e suas composições.

Decorator - Decorador (Objeto) Anexa dinamicamente responsabilidades adicionais a um objeto. Provê uma alternativa flexível ao uso de herança como modo de estender funcionalidade.

Facade - Fachada (Objeto) Provê uma interface unificada para o conjunto de interfaces de um subsistema. Define uma interface de alto nível que faz um subsistema mais fácil de usar. Flyweight - Peso-mosca (Objeto) Usa compartilhamento para suportar um grande número de pequenos objetos de forma eficiente.
